Personal Project Site Plan

Sam Bradshaw

WDD 131 Final Project Planning Document

Site Name

Sam Bradshaw Portfolio

This name works well because the site functions as a professional portfolio that presents my experience, selected projects, technical skills, and contact information in one place.

Purpose

The purpose of this site is to present a professional portfolio for Sam Bradshaw that highlights work experience, technical skills, and selected programming and data projects. The site gives employers, instructors, and collaborators a clear way to review my background, see the kind of work I have done, and contact me professionally.

Audience

The primary audience for this site is employers, recruiters, internship coordinators, and professional contacts who want to learn about my experience and technical abilities. A secondary audience includes classmates and instructors who may review my web development progress and project work.

Description of Dynamic Elements

JavaScript is used most heavily on the Projects page. Project information is stored in an array of objects and displayed dynamically in the page. Visitors can search projects by keyword and filter them by category such as web, Python, data, or school work.

The site uses multiple JavaScript functions, event listeners, DOM interaction, conditional logic, arrays, objects, and array methods such as filter, map, forEach, and some. JavaScript also powers a responsive mobile navigation menu and updates the current year in the footer.

Logo

The logo appears in the header and helps give the site a consistent identity across pages.

Design Rationale

The blue color palette creates a professional and trustworthy appearance. Merriweather gives headings a polished, established look, while Inter improves readability for paragraph text on both mobile and desktop screens.

Content

Home Page

The home page introduces me with a professional headline, a short summary, highlighted skills, and quick links to the projects and contact pages. It also includes a profile image to make the page more personal and visually engaging.

Experience Page

The experience page summarizes my work history, responsibilities, and measurable accomplishments. It helps visitors understand my professional background in business, coordination, communication, and analysis.

Projects Page

The projects page features project cards with titles, descriptions, categories, technologies, and links. This page includes interactive search and filter controls built with JavaScript so users can quickly find relevant projects.

Contact Page

The contact page provides my email, GitHub, and LinkedIn information. It also includes a professional goals section that explains the kinds of roles and collaboration I am interested in.

Technical Requirements Included

  • Semantic HTML structure with valid head elements on each page
  • Google Fonts loaded consistently across the site
  • Meta description, Open Graph tags, and favicon included in page heads
  • Responsive layout for mobile and desktop
  • Accessible skip link and keyboard-friendly navigation
